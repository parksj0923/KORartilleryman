네트워크 가상화를 이용하면 좋은점
하나의 ip를 이용하여 여러개의 가상 네트워크에 참여할 수 있다.

ex) 하나의 물리적인 서버의 ip는 192.168.32.101이라고 하자
 그위에 가상머신 1과 a가 올라가 있고, 1은 가상 네트워크 A, a는 가상네트워크 B에 연결되어 있다.
 이때 일반적인 네트워크라면 1과 a는 같은 ip를 사용하고 있기 때문에 충돌이 일어나 사용할 수 없다.
 하지만 가상 네트워크를 사용하기 때문에 1과 a는 충돌하지 않고 서로 간섭하지 않는다. 
 
 
VLAN 소개 
 네트워크 데이터 링크 계층 내 특정 필드를 이요하여 물리 스위치 장비에서 서로 다른 "네트워크"처럼 분리하여 관리
 
 모든 랜카드는 하나의 고유한 MAC Address를 가지고 있음 
 
 스위치 : OSI 모델의 데이터링크 계층에서 작동 
         물리적 포트에 연결된 기기에서 전송된 패킷을 받아 다시 보냄, 
         단, 패킷이 도달해야 하는 기기로 이어지는 포트를 통해서만 보냄
         
 ( 스위치 VS 허브 :
   허브 역시 리소스 공유를 목적으로 여러 기기를 하나로 연결할 수 있는데, 허브에 연결된 기기 집단을 LAN 세그먼트라고 한다.
   스위치와 다른점은 허브는 받은 패킷을 허브에 연결되 모든 기기로 브로드캐스팅한다.
   반면, 스위치는 패킷의 목적지 주소로 지정된 기기로 이어지는 포트로만 패킷이 전달된다. 
 
  스위치 VS 라우터 : 
   라우터는 트래픽 포워딩과 라우팅 기능을 제공하며, 라우터라는 이름도 여기서 유래
   라우터는 계층 3(네트워크계층)에서 작동하며 네트워크를 다른 네트워크로 연결하는데 사용
   쉽게 생각해서, LAN과 WAN의 차이
   기기는 스위치를 통해 로컬로 연결되고, 네트워크는 라우터를 통해 다른 네트워크에 연결된다. 
   패킷이 인터넷을 이르기까지 일반적으로 거치는 경로(기기>허브>스위치>라우터>인터넷)
   )
  
  포트가 1-7번까지 있는 스위치가 있다고 가정하자
  이때 VLAN이 없는 스위치는 1-7까지의 모든 포트들은 서로 네트워크 통신이 가능한 하나의 네트워크에 연결되어 있다.
  하지만 VLAN이 있는 스위치는 포트1:VLAN 1, 포트2:VLAN2,포트3:VLAN1, 포트4:VLAN2,포트5:VLAN3 등 이렇게 하나의 스위치에 여러개의 네트워크가 존재하게 만들 수 있다. 이것을 가상 네트워크라고 한다.
  
 VLAN: Trunk모드
 스위치1에서 가상 네트워크 뿐만아니라 스위치2에서도 같은 가상 네트워크를 만들어 서로 연결하고 싶을때
 -Trunk모드 비활성화일때 : 새로운 스위치3을 만들어 VLAN번호마다 포트를 연결시켜줘야함. 이 포트들을 통해서 스위치 1과 2를 연결함
 
 -Trunk모드 활성화 : 스위치1,2에 해당하는 포트만 있으면 됨
